# 智能头像构图

演员头像编辑器使用本地、确定性的人脸检测与关键点算法生成裁剪参数。图片不会发送给模型供应商，也不依赖“设置 → 模型”中的默认模型。

## 目标

- 支持按脸部或完整头部选择居中基准；默认脸部模式将双眼中心置于头像高度约 45% 的位置。
- 不同原图中的脸部视觉尺寸保持接近。
- 多人图片不静默猜测目标；候选人脸接近时由用户选择。
- 检测失败、原图留白不足或人脸靠近边缘时不破坏当前裁剪，仍可手动调整。
- Windows、macOS 和 Linux 使用同一模型、CPU delegate 与几何规则。

## 处理流程

1. Renderer 将当前原图等比缩小到最长边不超过 1536px，创建 `ImageBitmap`。
2. 图片通过 transferable 交给 module Web Worker，避免主线程执行模型推理。
3. MediaPipe BlazeFace Full Range 在整图检测人脸，置信度阈值为 0.65。
4. 整图没有结果时，使用四个 62% 尺寸的重叠分块补检；IoU 不低于 0.4 的重复结果合并。
5. 候选按相对面积 58%、置信度 27%、接近图片中心 15% 排序，最多保留五张脸。
6. 每个候选扩展为检测框的 1.75 倍正方形 ROI，并缩放到 384px；Face Landmarker 在 ROI 内提取双眼、脸顶、下巴和双颊关键点。
7. 关键点必须为有限值、能映射回原图且与检测框一致，否则降级使用 BlazeFace 检测框和眼睛关键点。
8. 选择“头部居中”时，Hair Segmenter 在候选人脸的局部 ROI 中识别真实头发轮廓；只取与当前人脸相连的头发区域，并排除下巴以下的长发和其他人物噪声。
9. 本地几何函数计算缩放与偏移，结果仍可拖动和缩放；保存逻辑与手动裁剪完全相同。

Worker 在第一次点击“智能构图”时懒加载。选择“头部”居中或开启“完整头部”保护时才加载和运行 Hair Segmenter；关闭保护的“脸部”居中不会承担头发分割开销。设置页的构图样张也使用同一检测服务与裁剪几何；拖动缩放滑杆时复用已经检测到的关键点，只重新计算裁剪参数。没有待处理请求后空闲 60 秒会销毁，释放模型和 WASM 内存。

MediaPipe 创建任务后会清空全局 `ModuleFactory`，而浏览器会缓存 ESM loader。Detector 与 Landmarker 因此使用带不同查询参数的 loader URL，确保工厂脚本分别执行；两者仍指向同一个 WASM 文件，不会增加打包体积。

## 构图规则

用户可以在“设置 → 外观 → 智能头像构图”进入编辑模式，分别设置居中基准、构图范围和完整头部保护。构图范围滑杆在界面中只显示“更宽松—更紧凑”，内部对应 50%–75% 的脸部椭圆高度占比。调整时只更新样张预览，点击“保存”后才写入设置；取消会恢复已保存值。设置只影响之后点击“智能构图”生成的默认结果，不会重裁已有头像。

设置样张首次打开时由本地模型检测完整的人脸和头部几何信息，检测结果会写入带版本的本地缓存。后续进入设置页直接使用缓存坐标计算预览，不重复加载模型，也不会先展示原图再出现一次可见的缩放。样张或检测逻辑发生变化时应更新缓存 key 版本。

居中方式：

| 模式 | 锚点规则 | 用途 |
|---|---|---|
| 脸部（默认） | 双眼中心位于头像高度 45% | 以五官为视觉中心，保持原有头像构图 |
| 头部 | 真实头发轮廓至下巴的中心位于头像高度 50% | 平衡发顶、下巴及左右两侧留白 |

Face Landmarker 不提供真实发顶关键点，因此头部居中额外使用官方 Hair Segmenter。算法把当前候选的头发掩码与脸部检测框、双颊和下巴合并为可见头部包围框，再同时按水平和垂直中心对齐。头发分割失败时才降级为脸部椭圆向上延伸 35% 的估算，并在界面明确提示用户手动微调。

“完整头部”保护是独立开关，默认关闭；开启时把发顶至下巴的头部框限制在头像框的 94% 以内，必要时缩小画面。该约束与居中基准无关：同一张图、同一缩放值和同一保护状态下，“脸部”和“头部”的 `baseScale` 与 `zoom` 必须完全一致，只有偏移不同。关闭保护后，缩放完全由滑杆决定。

构图范围滑杆内部范围为 50%–75%，步长为 1%，默认值为 50%。默认使用脸部居中，并关闭完整头部保护。旧版本保存的近景、标准、宽松挡位仍会在读取设置时分别迁移为 75%、70%、64%，已有用户配置不会被覆盖。

70% 校准参考使用以下目标比例：

| 几何量 | 在头像框中的目标比例 |
|---|---:|
| 双眼中心高度 | 45% |
| 双眼距离 | 25% |
| 脸部椭圆高度（脸顶至下巴） | 70% |
| 双颊距离 | 52% |

脸部椭圆是尺寸计算的主基准，权重为 60%。双眼与双颊各用于小幅校正，且相对椭圆基准最多偏移 18%，避免单个漂移关键点造成脸部忽大忽小。拖动滑杆时，椭圆、眼距、双颊和检测框目标按同一连续系数缩放；仅有检测框时，70% 校准参考对应的目标检测框高度为头像框的 74%。

最终缩放限制在 1–4 倍，偏移通过现有裁剪边界函数约束，保证正方形输出区域不会出现空白。受原图约束时 UI 会明确说明：

- `source-too-tight`：原图留白不足，无法继续缩小人脸；
- `source-edge`：人脸靠近边缘，无法完全移到目标位置；
- `zoom-limit`：人脸过小，达到 4 倍放大上限。

## 多人脸与失败策略

第二候选的显著度达到第一候选的 72% 时视为有歧义。编辑器显示编号按钮并等待用户选择，不自动改变当前构图。差距足够大时自动采用首选候选，同时保留编号按钮供用户改选。

模型初始化失败、超时、没有检测到清晰人脸或 Worker 异常时，只显示错误提示，当前裁剪参数保持不变。单次分析超时为 30 秒。

## 打包与运行影响

模型运行部分只携带 module-SIMD WASM、一个检测模型、一个关键点模型和一个轻量头发分割模型，不复制完整 npm 包或 no-SIMD 运行时。设置页另带一张高分辨率线稿样张，用于复用同一检测与裁剪流程生成预览。

| 资源 | 原始大小（约） |
|---|---:|
| module-SIMD WASM + loader | 10.94 MiB |
| BlazeFace Full Range | 1.03 MiB |
| Face Landmarker | 3.58 MiB |
| Hair Segmenter | 0.75 MiB |
| 模型运行资源小计 | 16.31 MiB |
| 设置预览样张 | 1.05 MiB |
| 原始资源合计 | 17.36 MiB |

安装包通常会压缩这部分资源；实际增量取决于目标格式和压缩器。`npm run build` 会运行 `scripts/check-mediapipe-assets.mjs`：校验三个模型的 SHA-256、确认只复制一个 WASM 变体，并阻止五个模型运行资源超过 17 MiB；设置预览样张由 Vite 作为普通图片资源单独打包。

推理耗时受 CPU、原图和人脸数量影响。首次点击包含 WASM/模型加载，明显慢于后续调用；“头部”居中会对每个候选增加一次 512×512 Hair Segmenter 推理，官方 Pixel 6 CPU 基准约为 57.90ms，桌面端实际耗时需单独测量。推理在 Worker 中进行，不阻塞拖动和页面交互。Worker 存活时会额外占用模型与推理内存，空闲销毁后释放。

## 验证清单

- 单人正脸、侧脸、带刘海、短发、长发、戴眼镜、低分辨率、小脸远景；
- 两人大小接近与主次明显的图片；
- 人脸贴近四条边、原图裁得过紧、横图和竖图；
- 连续切换候选、手动微调、重置、取消及保存后重新编辑；
- Windows x64、macOS Intel/Apple Silicon、Linux 的首次和热启动耗时；
- 断网状态首次使用（所有运行资源必须来自应用包）。
